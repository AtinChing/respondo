# File tree (empty scaffold map)

Snapshot of the **respondo** repo layout. Lines describe what each path is and what it holds. Dependency trees (`Frontend/node_modules/`, `Backend/.venv/`) and Git internals under `.git/objects/` are omitted for size; they are present on disk when you install dependencies or use Git.

```
respondo/
├── .cursor/                          — Cursor IDE folder (empty in this snapshot; optional editor/project config).
├── .git/                             — Git repository metadata, refs, and history (not expanded here).
├── .gitignore                        — Ignore rules for secrets, Python cache, venv, and Railtracks local data.
├── README.md                         — Short project pitch: security / emergency response / surveillance agent.
├── ASSISTANT_UI_DOCS.md              — Notes/docs for Assistant UI integration patterns used in the frontend.
├── RAILTRACKS_DOCS.md                — Notes/docs for Railtracks usage (aligned with backend tracing).
│
├── Backend/                          — Python FastAPI service: video upload, analysis API, Railtracks-wrapped vision agent.
│   ├── main.py                       — FastAPI app: CORS, `/health`, `/api/analyze-video` (multipart upload → analysis).
│   ├── video_agent.py                — OpenCV frame extraction + Railtracks agent; structured issue classification output.
│   ├── pyproject.toml                — uv/Python project metadata and dependency declaration.
│   ├── uv.lock                       — Locked dependency versions for reproducible installs with uv.
│   ├── requirements.txt              — Pip-style dependency list (alternate install path).
│   ├── sample.mov                    — Sample video asset for local testing of the pipeline.
│   ├── .env                          — Local API keys and config (gitignored; not committed).
│   ├── __pycache__/                  — Bytecode cache for `main` and `video_agent` (generated by Python).
│   └── .railtracks/                  — Local Railtracks data directory (gitignored).
│       └── data/sessions/            — Persisted session/trace JSON files from Railtracks runs (e.g. video frame description sessions).
│
└── Frontend/                         — React + Vite + TypeScript SPA; Assistant UI, AI SDK, issue dashboard and video analysis UI.
    ├── .gitignore                    — Frontend-specific ignore rules.
    ├── .env.local                    — Vite local env (e.g. API base URL; gitignored).
    ├── index.html                    — Vite HTML entry; mounts the React app.
    ├── package.json                  — npm scripts and runtime/dev dependencies (React 19, Vite 8, assistant-ui, ai SDK).
    ├── package-lock.json             — Locked npm dependency tree.
    ├── vite.config.ts                — Vite bundler config (React plugin, dev server, build options).
    ├── tsconfig.json                 — Root TypeScript compiler options.
    ├── tsconfig.app.json             — TS config scoped to application source.
    ├── tsconfig.node.json            — TS config for Node-side tooling (e.g. Vite config).
    ├── eslint.config.js              — ESLint flat config for the frontend.
    ├── README.md                     — Frontend-specific readme (Vite/React template notes).
    │
    ├── public/                       — Static assets copied to build output as-is.
    │   ├── favicon.svg               — Site favicon.
    │   └── icons.svg                 — SVG icon sprite or set used by the UI.
    │
    ├── dist/                         — Production build output from `vite build` (hashed JS/CSS, static assets).
    │   ├── index.html                — Built HTML shell referencing hashed assets.
    │   ├── favicon.svg               — Copied favicon.
    │   ├── icons.svg                 — Copied icons.
    │   └── assets/                   — Bundled JS and CSS (content-hashed filenames).
    │       ├── index-*.js            — Application JavaScript bundle.
    │       └── index-*.css           — Compiled styles.
    │
    └── src/                          — Application source.
        ├── main.tsx                  — React DOM root; mounts `<App />`.
        ├── App.tsx                   — Top-level routing between issues dashboard, video analysis, and settings via `AppShell`.
        ├── index.css                 — Global styles.
        ├── types.ts                  — Shared TypeScript types for the UI domain.
        │
        ├── components/               — React UI components.
        │   ├── AppShell.tsx          — Layout shell: navigation between app pages.
        │   ├── IssueDashboard.tsx    — Issues list/dashboard view.
        │   ├── IssueAgentChat.tsx    — Chat UI wired to assistant/AI for issue workflows.
        │   ├── ClassifyIssueDialog.tsx — Dialog for classifying or triaging an issue.
        │   ├── GetVidAnalysisPage.tsx — Page to upload/analyze video against the backend API.
        │   └── SettingsPage.tsx      — App settings UI.
        │
        ├── hooks/                    — Custom React hooks.
        │   ├── useIssues.ts          — Issue data fetching/state for the dashboard.
        │   └── useDepartments.ts     — Department-related data or options for the UI.
        │
        ├── data/                     — Static or seed data modules.
        │   └── issues.ts             — Issue records or mock/seed data for development and demos.
        │
        └── assets/                   — Images and static imports used by components.
            ├── hero.png              — Hero/marketing image.
            ├── react.svg             — React logo asset.
            └── vite.svg              — Vite logo asset.
```
